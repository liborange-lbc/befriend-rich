# 微众银行资产数据导入与自动分类

## 背景与动机

用户每周收到微众银行电子版资产对账单邮件（163 邮箱），包含所有基金持仓的名称和金额。目前需要手动处理这些数据。本需求旨在：
1. 实现一键导入微众银行资产数据到系统
2. 自动从邮箱定时拉取最新对账单
3. 自动识别并创建新基金、自动分类

数据源示例（webank_assets_20260417_234305.xlsx）：
- 35 条基金持仓记录
- 字段：资产项（基金名称）、金额(元)、币种（均为 CNY）
- 无基金代码，需通过 AkShare 自动查询

## 功能范围

### 包含

#### 子需求 1：Excel 手动导入
- 前端上传 WeBank Excel 文件（xlsx 格式）
- 后端解析 Excel，提取资产项、金额、币种
- 通过 AkShare API 根据基金名称自动查询基金代码
- 匹配系统已有 Fund 记录，不存在的自动创建
- 将持仓数据写入 PortfolioRecord 表
- 自动生成 PortfolioSnapshot

#### 子需求 2：邮箱自动拉取
- 后端集成 IMAP 邮箱拉取功能（参考 webank-statement-to-excel skill 的脚本逻辑）
- 从 163 邮箱自动拉取主题为"微众银行电子版资产对账单"的最新邮件
- 解密 zip 附件（密码 090391）→ 提取 PDF → 解析资产概览
- 前端提供"从邮箱拉取"按钮，手动触发
- APScheduler 定时任务：每天 9:00 自动拉取一次
- IMAP 凭据（邮箱账号、授权码）存储在 SystemConfig 表，前端设置页可配置
- 防重复导入：按日期判断，同一份对账单不重复录入

#### 子需求 3：自动建仓与 AI 分类
- Excel 中基金名称不在系统中时，自动创建 Fund 记录
  - 通过 AkShare 查询基金代码、补充基金元数据
  - 自动触发历史数据回填（backfill）
- 调用 Claude API 分析基金名称，自动决策每个 ClassModel 下的分类归属
  - 将系统中所有 ClassModel 及其 ClassCategory 树结构作为上下文传给 AI
  - AI 返回每个基金在每个模型下应归入的 category_id
  - 自动创建 FundClassMap 记录

#### 子需求 4：资产记录页面（新建独立页面）
- 新建前端页面"资产记录"
- 展示历史所有 PortfolioRecord 数据
- 顶部操作区：
  - "上传 Excel" 按钮：上传 xlsx 文件导入
  - "从邮箱拉取" 按钮：一键触发邮箱拉取并录入
- 数据过滤：
  - 日期范围筛选
  - 基金名称/代码搜索
  - 分类模型筛选
- 多维度自由分组：
  - 提供维度选择器，可选维度：日期（周/月）、分类模型各类别、币种等
  - 用户可自由选择任意维度组合
  - 分组后显示：总持仓金额、总收益
- 表格展示所有明细数据

### 不包含
- PDF 文件直接上传（仅支持 xlsx）
- 多邮箱支持（仅 163 邮箱）
- 基金分类的人工审核流程（全自动）
- 资产数据的编辑/删除功能（本期不做）

## 用户场景

- 场景 1：用户在"资产记录"页点击"上传 Excel"，选择本周的 webank_assets_xxx.xlsx 文件，系统自动解析、匹配/创建基金、AI 分类、录入持仓数据、生成快照，页面刷新显示最新数据
- 场景 2：用户在"资产记录"页点击"从邮箱拉取"，系统自动连接 163 邮箱、找到最新对账单邮件、解密解析、完成全流程录入
- 场景 3：每天 9:00 系统自动执行邮箱拉取任务，如有新的对账单则自动录入
- 场景 4：用户在"资产记录"页选择按"资产类别"分组，查看各类别的总持仓和总收益；切换为按"月份+地区"分组，查看不同月份不同地区的资产分布
- 场景 5：用户在设置页配置 163 邮箱账号和 IMAP 授权码

## 验收标准

- [ ] 上传 WeBank Excel 文件后，35 条持仓数据全部正确录入 PortfolioRecord
- [ ] 系统中不存在的基金自动通过 AkShare 查询代码并创建 Fund 记录
- [ ] 新创建的基金自动触发历史数据回填
- [ ] 新基金通过 Claude API 自动完成所有分类模型的分类归属
- [ ] "从邮箱拉取"按钮能成功连接 163 邮箱并完成全流程
- [ ] 定时任务每天 9:00 自动执行邮箱拉取
- [ ] 同一份对账单不会重复导入
- [ ] 设置页可配置 IMAP 邮箱凭据
- [ ] "资产记录"页正确展示历史数据
- [ ] 多维度自由分组功能正常，分组后显示总持仓金额和总收益
- [ ] 日期范围、基金搜索、分类筛选等过滤功能正常

## 非功能需求

- 性能：Excel 导入 35 条数据应在 10 秒内完成（不含 AI 分类和回填）
- 安全：IMAP 授权码在 SystemConfig 中存储，API 不返回明文密码
- 可靠性：邮箱拉取失败时记录错误日志，不影响系统正常运行
- 兼容性：支持 webank-statement-to-excel skill 产出的标准 xlsx 格式
