## 为什么

个人基金投资缺乏系统化的管理和分析工具。目前的痛点：
1. **资产分散**：持有中国和美国市场的多只基金，缺少统一视图和多维度分类分析（良田模型、股债比、全天候模型等）
2. **分析低效**：手动查看基金行情，无法系统性地追踪均线偏差和买卖时机
3. **策略无法验证**：投资策略停留在感觉层面，缺少回测验证和自动化提醒

现在做是因为投资组合已达到需要系统管理的规模（近百只基金），手动管理已不可持续。

## 变更内容

从零构建 **BeFriend FundAsset** 平台，包含：

- **新增** Web 应用（React + FastAPI），提供可视化大盘看板、资产管理、基金分析、回测四大模块
- **新增** 可自定义的多维度资产分类模型系统（良田模型、股债比、全天候等，层级数量可定义）
- **新增** 每周持仓录入与资产趋势追踪（总金额 + 收益预留字段）
- **新增** 行情数据自动抓取服务（Tushare + Yahoo Finance，每小时运行，同日覆盖）
- **新增** 技术指标计算引擎（MA30/60/90/120/180/360 + 均值偏差）
- **新增** 轻量回测引擎（定投、条件买卖、止盈止损，输出收益率/夏普比率/最大回撤等）
- **新增** 可保存策略 + 定时策略检查（每日 9/12/14 点）+ 飞书机器人通知
- **新增** 汇率自动获取与多币种资产折算（USD/CNY）
- **新增** Docker Compose 部署方案

## 功能 (Capabilities)

### 新增功能

- `fund-management`: 基金/指数的录入管理（代码、名称、数据源、费率、币种、启停状态）
- `classification-model`: 可自定义的多维度资产分类模型（N级层级，多模型并存，基金↔分类映射）
- `portfolio-tracking`: 每周持仓录入（基金+总金额+收益）、资产快照、趋势追踪
- `market-data`: 行情数据抓取服务（多数据源适配：Tushare/Yahoo Finance，定时调度，汇率获取）
- `technical-analysis`: 技术指标计算（MA均线、均值偏差）与可视化分析
- `backtest-engine`: 回测引擎（策略配置、历史回测、指标计算、交易日志）
- `strategy-alert`: 策略管理与提醒（可保存策略、定时检查、条件触发、飞书通知）
- `dashboard`: 大盘看板（行情总览、K线图、均值偏差热力图、策略提醒、资产概览）
- `asset-report`: 资产报表页（总资产、多模型饼图、分类趋势图、TOP5持仓榜）

（无，全新项目）

## 影响

- **全新代码库**：前端 React + TypeScript + Ant Design + ECharts，后端 Python + FastAPI + SQLAlchemy
- **外部依赖**：Tushare API（需 token）、Yahoo Finance API、飞书机器人 API（需 app_id/secret）
- **数据库**：SQLite，通过 Docker volume 持久化
- **定时任务**：APScheduler 运行行情抓取和策略检查
- **部署**：Docker Compose（frontend nginx + backend + sqlite volume）
