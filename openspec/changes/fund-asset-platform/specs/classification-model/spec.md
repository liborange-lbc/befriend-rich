## 新增需求

### 需求:自定义分类模型
系统必须支持用户创建多个资产分类模型（如良田模型、股债比模型、全天候模型）。每个模型有唯一名称和描述。

#### 场景:创建分类模型
- **当** 用户提交模型名称和描述（如"良田模型"）
- **那么** 系统创建分类模型记录

#### 场景:模型名称唯一性
- **当** 用户创建模型时使用已存在的名称
- **那么** 系统拒绝创建并返回"模型名称已存在"错误

#### 场景:编辑分类模型
- **当** 用户修改模型名称或描述
- **那么** 系统更新模型信息

#### 场景:删除分类模型
- **当** 用户删除一个分类模型
- **那么** 系统删除该模型及其所有类别和基金映射关系

### 需求:多级类别管理
系统必须支持在每个分类模型下创建多级树形类别结构。层级数量不限，每个类别有名称、所属模型、父类别（可为空表示顶级）和排序序号。

#### 场景:创建顶级类别
- **当** 用户在"良田模型"下创建类别"粮食作物"，不指定父类别
- **那么** 系统创建一级类别"粮食作物"

#### 场景:创建子类别
- **当** 用户在"粮食作物"下创建子类别"中国宽基"
- **那么** 系统创建二级类别"中国宽基"，父类别为"粮食作物"

#### 场景:编辑类别
- **当** 用户修改类别名称或排序
- **那么** 系统更新类别信息

#### 场景:删除类别
- **当** 用户删除一个类别
- **那么** 系统删除该类别及其所有子类别，同时删除相关的基金映射

### 需求:基金分类映射
系统必须支持将基金映射到不同分类模型的类别下。一只基金在同一个模型中只能映射到一个叶子类别，但可以在不同模型中各有一个映射。

#### 场景:映射基金到类别
- **当** 用户将"沪深300ETF"映射到良田模型的"粮食作物/中国宽基"类别
- **那么** 系统创建映射关系

#### 场景:同模型重复映射
- **当** 用户将同一基金在同一模型下映射到第二个类别
- **那么** 系统替换原有映射（一个基金在一个模型中只有一个分类）

#### 场景:跨模型映射
- **当** 用户将"沪深300ETF"分别映射到良田模型的"粮食作物/中国宽基"和股债模型的"股"
- **那么** 系统保存两条独立的映射关系

#### 场景:查看基金分类
- **当** 用户查看某只基金的分类信息
- **那么** 系统返回该基金在所有模型中的分类映射列表
