## 新增需求

### 需求:行情数据抓取
系统必须支持通过可扩展的数据源适配器自动抓取基金/指数的每日收盘价。

#### 场景:定时抓取
- **当** 调度器按每小时触发抓取任务
- **那么** 系统遍历所有启用状态的基金，通过其指定的数据源适配器获取当日收盘价

#### 场景:同日数据覆盖
- **当** 同一日内多次抓取同一基金数据
- **那么** 系统用最新数据覆盖当日已有记录

#### 场景:抓取失败处理
- **当** 某只基金的数据源请求失败
- **那么** 系统记录错误日志，跳过该基金，继续处理其他基金，不影响整体抓取流程

### 需求:数据源适配器
系统必须实现统一的数据源接口，当前支持 Tushare（A股/国内基金）和 Yahoo Finance（美股/海外）。未来必须可通过实现相同接口扩展新数据源。

#### 场景:Tushare 适配器
- **当** 基金数据源类型为 tushare
- **那么** 系统使用 Tushare API 获取该基金的每日收盘价数据

#### 场景:Yahoo Finance 适配器
- **当** 基金数据源类型为 yahoo
- **那么** 系统使用 yfinance 库获取该基金的每日收盘价数据

#### 场景:新增数据源
- **当** 开发者实现新的数据源适配器（如 AKShare）
- **那么** 只需实现统一接口并注册，无需修改上层业务逻辑

### 需求:汇率数据获取
系统必须自动获取每日汇率数据（至少 USD/CNY），用于多币种资产折算。

#### 场景:获取每日汇率
- **当** 调度器触发汇率抓取任务
- **那么** 系统获取当日 USD/CNY 汇率并存储

#### 场景:汇率数据缺失兜底
- **当** 当日汇率获取失败
- **那么** 系统使用最近一个有效交易日的汇率作为兜底值

### 需求:历史数据回填
系统必须支持手动触发历史数据回填，用于初次设置时获取基金的历史价格数据。

#### 场景:回填历史数据
- **当** 用户对某只基金触发历史数据回填，指定开始日期
- **那么** 系统从指定日期到当前日期批量获取并存储所有交易日的收盘价数据
