## 新增需求

### 需求:策略保存与管理
系统必须支持用户保存、编辑、启停和删除投资策略。策略包括名称、关联基金（可选）、策略类型、策略配置参数和提醒条件。

#### 场景:保存策略
- **当** 用户从回测页面保存一个策略配置
- **那么** 系统持久化策略记录，默认启用提醒

#### 场景:编辑策略
- **当** 用户修改已保存策略的参数或提醒条件
- **那么** 系统更新策略配置

#### 场景:启停策略
- **当** 用户关闭某策略的提醒开关
- **那么** 该策略不再参与定时检查，但策略配置保留

#### 场景:删除策略
- **当** 用户删除一个策略
- **那么** 系统删除策略及其关联的回测结果

### 需求:定时策略检查
系统必须在每日 9:00、12:00、14:00 自动运行所有启用状态的策略检查。

#### 场景:定时触发检查
- **当** 到达检查时间点（9:00/12:00/14:00）
- **那么** 系统遍历所有启用的策略，使用最新行情数据评估提醒条件

#### 场景:条件触发
- **当** 某策略的提醒条件被满足（如均值偏差突破阈值）
- **那么** 系统生成提醒消息并通过飞书机器人发送通知

#### 场景:条件未触发
- **当** 策略检查完成但无条件被满足
- **那么** 系统不发送通知，记录检查日志

### 需求:提醒条件可定义
系统必须支持用户自定义提醒条件，条件基于技术指标（均值偏差、MA 交叉等）的比较表达式。

#### 场景:配置偏差阈值条件
- **当** 用户设置条件"MA60 偏差 < -8%"
- **那么** 系统在 MA60 偏差低于 -8% 时触发提醒

#### 场景:配置多条件组合
- **当** 用户设置多个条件并指定"全部满足"或"任一满足"
- **那么** 系统按逻辑关系评估条件组合

### 需求:飞书机器人通知
系统必须通过飞书自建应用机器人发送策略提醒消息。消息必须包含基金名称、触发条件、当前指标值和建议操作。

#### 场景:发送飞书通知
- **当** 策略条件被触发
- **那么** 系统通过飞书机器人 API 发送富文本卡片消息，包含基金名称、触发条件描述、当前数值

#### 场景:飞书发送失败
- **当** 飞书 API 调用失败
- **那么** 系统记录错误日志并重试一次，仍失败则跳过
