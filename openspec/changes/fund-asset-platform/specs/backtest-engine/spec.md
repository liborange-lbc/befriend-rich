## 新增需求

### 需求:回测策略配置
系统必须支持用户配置回测策略，包括买入策略、卖出策略和回测时间范围。

#### 场景:配置定投策略
- **当** 用户配置定投策略：每周/每月定额投入固定金额
- **那么** 系统保存策略配置，支持后续回测运行

#### 场景:配置条件买入策略
- **当** 用户配置条件买入：当均值偏差低于指定阈值时买入
- **那么** 系统保存策略配置

#### 场景:配置卖出策略
- **当** 用户配置卖出策略：止盈（收益率达标）、止损（亏损超限）、条件卖出（均线信号）
- **那么** 系统保存策略配置

#### 场景:配置回测时间范围
- **当** 用户指定回测起止日期（如最近5年或10年）
- **那么** 系统使用该时间范围内的历史数据运行回测

### 需求:回测执行
系统必须基于历史价格数据和策略配置模拟交易过程，生成完整的交易日志和净值曲线。

#### 场景:运行回测
- **当** 用户选择一只基金和一个策略配置，点击运行回测
- **那么** 系统基于历史数据模拟交易过程，生成结果

#### 场景:回测结果指标
- **当** 回测运行完成
- **那么** 系统计算并返回以下指标：总收益率、年化收益率、夏普比率、最大回撤、波动率、胜率、盈亏比

#### 场景:基准对比
- **当** 回测结果生成后
- **那么** 系统同时计算同时间段内基准指数（如沪深300）的买入持有收益，作为对比基准

#### 场景:交易日志
- **当** 回测运行完成
- **那么** 系统返回每笔模拟交易的明细（日期、买/卖、金额、价格、持仓、收益）

### 需求:回测结果可视化数据
系统必须输出可供前端绘制的回测结果数据，包括净值曲线和关键指标。

#### 场景:净值曲线数据
- **当** 前端请求回测结果图表数据
- **那么** 系统返回按日期排列的策略净值序列和基准净值序列

#### 场景:回撤曲线数据
- **当** 前端请求回撤图表数据
- **那么** 系统返回按日期排列的回撤百分比序列
