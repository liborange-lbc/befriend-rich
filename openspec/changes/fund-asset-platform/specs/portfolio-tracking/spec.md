## 新增需求

### 需求:持仓录入
系统必须支持用户按周录入每只基金的持仓数据，包括基金、录入日期、总金额（本币）和收益（预留字段）。

#### 场景:录入单只基金持仓
- **当** 用户选择基金"沪深300ETF"，输入日期"2026-04-18"，金额 50000，收益 3200
- **那么** 系统创建持仓记录

#### 场景:批量录入持仓
- **当** 用户在录入页面一次提交多只基金的持仓数据
- **那么** 系统批量创建所有持仓记录

#### 场景:修改已录入的持仓
- **当** 用户修改某日期某基金的已录入金额
- **那么** 系统更新该持仓记录并重新计算相关快照

#### 场景:同基金同日期去重
- **当** 用户对同一基金在同一日期重复录入
- **那么** 系统更新已有记录而非创建重复记录

### 需求:资产快照生成
系统必须在每次录入完成后自动生成资产快照，记录该日期的总资产（人民币折算）和各分类模型的分类汇总数据。

#### 场景:生成快照
- **当** 用户完成一次持仓录入
- **那么** 系统自动计算并保存：总资产（人民币）、每个分类模型各类别的金额汇总

#### 场景:汇率折算
- **当** 快照生成时存在 USD 币种的基金
- **那么** 系统使用当日 USD/CNY 汇率将 USD 金额折算为人民币后汇总

### 需求:持仓历史查询
系统必须支持查询历史持仓记录和快照数据，用于趋势分析。

#### 场景:查询持仓历史
- **当** 用户查询某个时间范围内的持仓记录
- **那么** 系统返回该时间范围内所有录入记录，按日期排序

#### 场景:查询快照趋势
- **当** 用户查询资产趋势数据
- **那么** 系统返回所有快照数据（日期、总资产、各模型分类汇总），用于绘制趋势图
