## 新增需求

### 需求:均线计算
系统必须基于每日收盘价计算 MA30、MA60、MA90、MA120、MA180、MA360 六条均线，并将结果存储到每日价格记录中。

#### 场景:计算均线
- **当** 系统获取到新的每日收盘价数据
- **那么** 系统自动计算该基金在该日期的所有均线值（需要足够的历史数据量）

#### 场景:历史数据不足
- **当** 某只基金的历史数据不足以计算某条均线（如不足 360 天无法计算 MA360）
- **那么** 该均线值记为空（null），其他可计算的均线正常输出

### 需求:均值偏差计算
系统必须计算每日收盘价与各均线的偏差百分比：deviation = (close - MA) / MA × 100%。

#### 场景:计算偏差
- **当** 系统计算完均线后
- **那么** 系统自动计算收盘价与每条均线的偏差百分比并存储

#### 场景:均线为空时的偏差
- **当** 某条均线值为空
- **那么** 对应的偏差值也为空

### 需求:技术指标查询
系统必须支持按基金和时间范围查询技术指标数据（收盘价、均线、偏差），用于前端可视化展示。

#### 场景:查询单基金指标
- **当** 用户选择一只基金和时间范围
- **那么** 系统返回该时间范围内的每日收盘价、各均线值和各偏差值

#### 场景:查询多基金偏差汇总
- **当** 用户请求均值偏差热力图数据
- **那么** 系统返回所有启用基金的最新日期的各均线偏差值
