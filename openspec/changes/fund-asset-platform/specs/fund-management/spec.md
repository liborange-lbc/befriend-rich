## 新增需求

### 需求:基金录入与管理
系统必须支持用户录入和管理基金/指数信息，包括代码、名称、数据源、费率、币种和启停状态。每只基金的代码在系统内必须唯一。

#### 场景:创建新基金
- **当** 用户提交基金信息（代码、名称、数据源类型、费率、币种）
- **那么** 系统创建基金记录并返回基金详情

#### 场景:基金代码唯一性校验
- **当** 用户创建基金时使用已存在的基金代码
- **那么** 系统拒绝创建并返回"基金代码已存在"错误

#### 场景:编辑基金信息
- **当** 用户修改已有基金的名称、费率、数据源等信息
- **那么** 系统更新基金记录并返回更新后的详情

#### 场景:停用基金
- **当** 用户将基金状态设为停用
- **那么** 该基金不再参与行情抓取和策略检查，但历史数据保留

#### 场景:基金列表查询
- **当** 用户访问基金管理页面
- **那么** 系统返回所有基金列表，支持按名称/代码搜索、按状态筛选、分页展示

### 需求:数据源类型管理
系统必须支持多种数据源类型（tushare、yahoo），每只基金必须指定一个数据源类型用于行情抓取。

#### 场景:指定数据源
- **当** 用户创建或编辑基金时选择数据源类型
- **那么** 系统保存数据源类型，后续行情抓取使用对应的数据源适配器

### 需求:币种管理
系统必须支持多币种（CNY、USD），每只基金必须指定币种。非人民币基金在资产汇总时必须按当日汇率折算为人民币。

#### 场景:设置基金币种
- **当** 用户创建基金时选择币种为 USD
- **那么** 系统记录币种为 USD，在资产汇总计算时自动使用 USD/CNY 汇率折算
