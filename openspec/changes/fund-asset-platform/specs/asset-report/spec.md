## 新增需求

### 需求:资产总览展示
资产管理页面必须展示最近一次录入的资产总规模（人民币），以及与上一次录入的对比变化。

#### 场景:展示资产总览
- **当** 用户访问资产管理页面
- **那么** 页面顶部展示：总资产金额、较上次变化金额和百分比、最近录入日期

### 需求:多模型饼图
资产管理页面必须并排展示每个已定义分类模型的资产占比饼图。

#### 场景:展示模型饼图
- **当** 用户查看资产管理页面
- **那么** 页面展示所有分类模型的饼图（如良田模型、股债模型、全天候模型），每个饼图显示该模型下各一级类别的金额和占比

#### 场景:无映射基金处理
- **当** 某些基金未映射到某个分类模型
- **那么** 饼图中展示"未分类"类别，汇总未映射基金的金额

### 需求:分类趋势图
资产管理页面必须展示每个分类模型下各大类的资产趋势图（按录入时间展示历史变化）。

#### 场景:展示趋势图
- **当** 用户查看资产趋势区域
- **那么** 页面展示：总资产趋势图，以及每个分类模型下各一级类别的趋势图

#### 场景:趋势图数据粒度
- **当** 趋势图展示数据
- **那么** 每个数据点对应一次持仓录入日期，X 轴为时间，Y 轴为金额

#### 场景:时间范围控制
- **当** 用户通过滑块调整时间范围
- **那么** 趋势图更新显示范围

### 需求:持仓TOP5榜单
资产管理页面必须展示持仓金额最大的前5只基金。

#### 场景:展示TOP5
- **当** 用户查看资产管理页面
- **那么** 页面展示持仓 TOP5 列表：排名、基金名称、持仓金额、占总资产百分比、收益金额（如有）

### 需求:持仓录入入口
资产管理页面必须提供快速录入本周持仓数据的入口。

#### 场景:打开录入表单
- **当** 用户点击"录入本周数据"按钮
- **那么** 弹出录入表单，预填上次录入的基金列表和金额，用户可修改金额和收益

#### 场景:预填上次数据
- **当** 录入表单打开时
- **那么** 表单自动加载上次录入的所有基金及其金额作为默认值，用户只需修改有变化的项
